    Calculating the value of each hand

    • I calculate the value of a hand by using a double ended queue. The reason for using this data structure is for handling the "Ace" which can have a value of 1 or 11. If an "Ace" is dealt it is placed at the end of the queue while all other cards are inserted at the front. This particular detail for appending the "Ace" at the end of the queue is important when it comes to looping over the queue to calculate the hand total. By adding up all the cards before the "Aces" determines whether the "Ace" needs to be valued as an 11 or 1.

    • The "calculatePlayersHand()" function is called every time a card is added to the hand. The role of the function is to calculate the total value of the hand by adding up all the card values in the queue. The edge case arises when the total value of a hand is greater than 21. If the hand has "Aces" then the total value is subtracted by 11 and the "Aces" alternate value of 1 is added. If the result of the hand is below 21 after treating the "Ace" as a value of 1 then the hand total becomes the new result.
